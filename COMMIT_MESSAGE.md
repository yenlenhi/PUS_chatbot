Fix Railway deployment issues - PostgreSQL dialect and Redis support

## Changes Made

### 1. Fixed SQLAlchemy PostgreSQL Dialect Error
- **File:** `config/settings.py`
- **Issue:** Railway provides DATABASE_URL with `postgres://` scheme
- **Fix:** Auto-convert to `postgresql://` for SQLAlchemy 1.4+ compatibility
- **Impact:** Resolves "Can't load plugin: sqlalchemy.dialects:postgres" error

### 2. Added Railway Redis URL Support
- **File:** `config/settings.py`
- **Change:** Parse `REDIS_URL` or `REDIS_PRIVATE_URL` from Railway
- **Impact:** Supports Railway's Redis plugin out-of-the-box
- **Fallback:** Uses individual env vars if REDIS_URL not available

### 3. Updated Redis Connection with Password Support
- **Files:** 
  - `src/services/cache_service.py` - Added password parameter
  - `src/services/embedding_service.py` - Pass password to CacheService
- **Impact:** Supports secured Redis connections (Railway default)

### 4. Created Deployment Documentation
- **RAILWAY_FIX_GUIDE.md** - Comprehensive deployment guide
- **RAILWAY_QUICK_FIX.md** - Quick reference for fixes
- **test_railway_config.py** - Test script to verify configuration

## Testing


## Railway Environment Variables Required

```bash
# Auto-generated by Railway
DATABASE_URL=postgresql://...  # PostgreSQL connection
REDIS_URL=redis://...          # Redis connection (if Redis plugin added)

# Required - set manually
LLM_PROVIDER=gemini
GEMINI_API_KEY=your_api_key

# Optional
HF_TOKEN=your_huggingface_token  # For Vietnamese embedding model
ENABLE_REDIS_CACHE=true          # Enable/disable Redis cache
```

## Deployment Steps

1. Commit and push changes
2. Railway auto-deploys
3. Add Redis plugin (optional but recommended)
4. Set GEMINI_API_KEY environment variable
5. Monitor logs for successful startup
